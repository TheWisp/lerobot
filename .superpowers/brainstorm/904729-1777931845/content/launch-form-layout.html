Where does the Cloud option live in the launch form?

Both options share the same Model tab. The question is how visible the Local↔Cloud switch should be.

A

Top-of-form radio: Local | Cloud (Nebius)

Single "Train" button on the Model tab opens one form. First field is a radio that switches the rest of the form between local fields (output dir) and cloud fields (Nebius profile, push-to-Hub repo). Common fields (dataset, policy, hyperparams) appear once and apply to both.

B

Two buttons: Train locally | Train on Nebius

Two side-by-side buttons in the model detail panel, each opens a separate form pre-scoped to that target. No mode switch inside the form — fewer conditionals in the UI, but cloud-specific fields are only discoverable after clicking the cloud button.

Model detail panel — actions row
C

Subtab: Train  →  Local | Cloud nav

Open a "Train" view that occupies the full detail panel, with its own internal nav for Local / Cloud / (future) Resume. More room for cloud-specific UI like the cost meter, VM provisioning progress, and post-training cleanup actions — none of which fit comfortably into a small form.

…

My recommendation: A. The radio is a single visible affordance, common fields stay common, and the form is small enough that the Local↔Cloud branch only adds ~3 fields. C is over-engineered for one extra mode; B hides the cloud option behind a separate button which makes "what's the difference" less obvious.
