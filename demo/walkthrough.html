LeRobot Studio — Feature Walkthrough

A single GUI for collecting, editing, visualizing, teleoperating, and training robot-learning data.

Data editing URDF + trajectory Latency monitoring VR teleoperation Hub transfers Training Cloud GPUs on Nebius

Data editing

Curate datasets in place — trim episodes, drop bad frames, add reward/quality columns — with a review-before-apply workflow so nothing is destructive by surprise.

URDF visualization + trajectory

See the robot, not just numbers — a live URDF view in the data tab, the commanded-vs-actual "ghost" overlay, and the end-effector path drawn as a 3D tube across an episode.

Latency monitoring

Know exactly where every millisecond goes — a per-stage Gantt of the teleop / record / inference loop, with p50/p95, overrun ratio, and per-camera staleness.

VR teleoperation video

Drive the arms from a Quest 3 headset — WebXR Cartesian control, straight into the same record pipeline. Works against real hardware or a virtual robot.

Hub transfers

Upload/download datasets and models to the Hugging Face Hub in the background — a persistent tray with live progress, retry, and cancel, so a big transfer never blocks the UI.

Training

Launch a training run from the GUI on your workstation or a remote GPU, and watch it live — real progress and auto-parsed loss/lr pulled straight from the trainer's output.

Cloud GPUs on Nebius

Train on a Nebius GPU without leaving the GUI: it spawns a VM when a run starts, trains over SSH, and destroys it automatically when the run ends — no Nebius console, no forgotten instances. A TTL and a cloud-init poweroff are the cost backstops; teardown is verified after every run.
